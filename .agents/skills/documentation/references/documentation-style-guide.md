# Documentation Style Guide

## Writing

- Keep language concise and action-oriented.
- Prefer concrete commands and examples over abstractions.

## Consistency

- Keep `README.rst`, `CONTRIBUTING.md`, and `doc/` aligned.
- Update all places when commands or workflows change.

## Safety

- Verify referenced files and paths exist.
- Avoid stale examples that differ from current repo commands.

## Review

- Include what changed, why, and any impact on contributors/operators.
